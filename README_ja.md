# E-Ink Spectra 6 画像コンバーター

このスクリプトは、6色のe-inkスクリーンに表示するために画像を変換し、そのようなディスプレイに最適化します。PhotoPainter フォトフレーム（<https://www.waveshare.com/PhotoPainter-B.htm>）を購入しましたが、付属の変換スクリプトに満足できなかったため、このバージョンを作成しました。

## 目的

e-inkスクリーンには独特の色制限と特性があります。このコンバーターは、標準的な画像ファイル（JPG、PNG、HEICなど）を処理し、Waveshare 7.3インチ E6 フルカラー E-Paper e-inkディスプレイに適した形式に変換することを目的としています。

## 機能

* **色量子化**: 画像のカラーパレットをe-inkスクリーンがサポートする6色に削減します。
* **ディザリング**: ディザリングアルゴリズムを適用して中間色をシミュレートし、知覚できる詳細を改善します。
* **アトキンソンディザリングの追加**: フロイド–スタインバーグディザリングとは異なり、より局所的なディザーを持ち、視覚的により美しい画像を生成します。処理速度はかなり遅くなります。
* **リサイズとクロップ**: アスペクト比を維持しながら、e-inkディスプレイの解像度（1600x1200 または 1200x1600）に合わせて画像サイズを調整します。
* **画像補正**: 明るさ、コントラスト、彩度などの調整を適用して、e-inkディスプレイ上で画像をより鮮やかで鮮明に見せます。rgb_dist と luma_dist の相対的な寄与度を調整し、色相誤差をより重要視しています。距離メトリックの重み補正もチューニングしています。
* **HEICサポート**: HEICイメージファイルの処理を可能にします。
* **プログレスバー**: フォルダ処理時に表示されます。特に多数のファイルを処理する際に便利です。

## ソース例

このスクリプトは、/inspirationにある2つの既存スクリプト `convert-program1.py`（出典: <https://www.waveshare.com/wiki/PhotoPainter_(B)#Picture_Production）と> `convert-program2.py`（出典: <https://github.com/myevit/PhotoPainter_image_converter）を組み合わせて拡張したものです。>

## 使い方

ビルド済みのWindows用 .exe ファイルが提供されています。画像ファイルまたはフォルダをドラッグ＆ドロップするだけで使用できます。

1. **依存関係のインストール**:

    ```bash
    pip install -r requirements.txt
    ```

2. **単一画像の変換**:

    ```bash
    python ConvertTo6ColorsForEInkSpectra6.py path/to/your/image.jpg
    ```

3. **ディレクトリ内のすべての画像を変換**:

    ```bash
    python ConvertTo6ColorsForEInkSpectra6.py path/to/your/images/directory
    ```

4. **特定のオプションを使用**:

    ```bash
    python ConvertTo6ColorsForEInkSpectra6.py images/my_photo.jpg --mode cut --contrast 1.5 --saturation 1.1 --dither 3
    ```

## 実装の詳細

### アトキンソンディザリング

現在のアトキンソンディザリング実装は、速度のために簡略化された前方のみの誤差分散アプローチを使用しています。誤差は未処理のピクセル（右方向と下方向）にのみ分散され、以下の重みを使用します：

* 右: 1/8
* 左下: 1/8  
* 下: 1/4
* 右下: 1/8

合計誤差分散は5/8となり、これはアトキンソンディザリングの特徴です。前方のみのアプローチにより、誤差拡散のために画像全体を保存する必要がなくなるためメモリ効率が向上し、シングルパス処理が可能になります。これは、通常6ピクセル周辺（右々と下々の位置を含む）にそれぞれ1/8の重みで誤差を分散する標準的な6ピクセルアトキンソンアルゴリズムとは異なります。

### 色距離メトリック

カスタムカラーパレットの追加を検討しました。モニターの原色のような赤 #FF0000、黄色 #FFFF00、緑 #00FF00、青 #0000FF を想定するのではなく、ディスプレイ上の実際の色はそれよりも落ち着いており、特に青と緑はかなりシフトしています（参考: <https://forums.pimoroni.com/t/what-rgb-colors-are-you-using-for-the-colors-on-the-impression-spectra-6/27942）。これを満足のいく形で機能させることができなかったため、RGB> と輝度成分を慎重に調整した重みで組み合わせたカスタム距離メトリックを使用する色量子化アルゴリズムで補正することにしました：

* **カスタム輝度計算**: 標準的な輝度の代わりに、重み付けされた計算式（R×250 + G×350 + B×400）を使用しており、e-inkディスプレイに重要な緑と青の感度を強調しています。

* **RGB距離の重み**: RGB成分距離には、非対称な重み（R:0.250, G:0.350, B:0.400）と全体スケーリング係数0.75を使用しており、青の視認性を高め、ディスプレイの制限を補正するために赤の強調を軽減しています。

* **色相精度の優先**: 合計距離式（1.5×rgb_dist + 0.60×luma_dist）は、輝度差よりもRGB差を重視（1.5 対 0.60）しており、明るさの誤差よりも色相誤差をより重要とします。これにより、色のシフトを防ぎ、限られたパレットで色の忠実度を維持します。

* **人間の目の補正**: 重みは、異なる色に対する人間の目の感度（緑への感度が高く、赤への感度が低い）と、e-ink Spectra 6ディスプレイの特定の色レンダリング特性を考慮しています。

## 今後の改善点

* **色シフトの問題**: 補正されたパレットにはモニターの原色がアンカーとして含まれるようになりましたが、（アトキンソンを使用した場合）色量子化の性質上、高度に飽和した色が色相シフトすることがあります。パレットのさらなる微調整や代替量子化アルゴリズムの探索により改善できる可能性があります。例えば、非常に飽和した赤 `#FF0000`（RGB: 255, 0, 0）は、補正された赤 `#a02020`（RGB: 160, 32, 32）と数値的に大きく異なります。距離は大きいです。同時に、補正された青 `#5080b8`（RGB: 80, 128, 184）も遠いですが、元の画像の赤の正確な色合いによっては、青への距離が赤への距離よりも小さくなる可能性があります。距離計算に知覚的な色空間（CIELABなど）を調査すると、より良い結果が得られるかもしれません。
* **その他のディザリング方法**: 例: Joel Yliluomaの任意パレット位置ディザリングアルゴリズム（<https://bisqwit.iki.fi/story/howto/dither/jy/）>
